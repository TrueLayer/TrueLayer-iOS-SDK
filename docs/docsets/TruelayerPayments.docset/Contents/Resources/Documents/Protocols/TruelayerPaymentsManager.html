TruelayerPaymentsManager

public protocol TruelayerPaymentsManager

It describes the interface of the TruelayerPayment Manager APIs

We suggest to grab a concrete implementation of this protocol by using the .shared static property of the TruelayerPayments.Manager type. Then invoke the start method as soon as possible. The best place to do this is in the AppDelegate, SceneDelegate, or in the object responsible to initialize all the dependencies of your app.

  • start(environment:style:)

    It starts the SDK, configuring the environment.

    This method should be called only once, before actually using the SDK. Our suggestion is to obtain the TruelayerPayments.Manager.shared object and call start immediately.

    Note

    this method is thread-safe.

    Throws

    Throws:

    • TruelayerPayments.Error.sdkAlreadyStarted: if the app tries to start the SDK more than once
    • TruelayerPayments.Error.failedToStartSDK: if something went wrong when trying to start the SDK

  • processPayments(context:callback:)

    It presents the SDK in the app to carry out a payment.

    This method can be called multiple times to process different payments. Before using this method, make sure that the SDK has been started, otherwise it will raise an error.

    Note

    this method is thread-safe.

    Throws

    Throws:

    • TruelayerPayments.Error.sdkNotStartedYet: if the app tries to process a payment before starting the SDK.
