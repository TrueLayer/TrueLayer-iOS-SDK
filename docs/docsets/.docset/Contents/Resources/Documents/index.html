IOS Sample Payments App

A sample app integrated with TrueLayers payments-api.

Intended for example purposes only, to give an idea of the integration flow for TrueLayers Payments API V2.

For more information about Payments API integration you can browse our 📖 documentation.



Prerequisites

Redirect URI

You’ll need to add the deep link below to your Allowed redirect URIs TrueLayer settings.

  • com.tl-iospaymentsdemo://

Sample Backend API

For best practices we’ve setup an sample backend api extracting logic that should be done on your backend. For example
handling authentication with the payments-api.

Get this up & running via providing your client_id, client_secret and your test bank details. And you’ve got a backend ready for testing. ✨


Project Setup 🚀

Developed with Xcode 12.4

  • Clone the project onto your local machine.

Dependencies setup

  • Once cloned navigate to the project directory.
  • Run setup-developer script. This will install all of the necessary development tools - git pre hook, swiftlint, swiftformat. ./bin/setup-developer


Configuration

  • Update your projects Configuration file Sample Backend uri provided from the prerequisites steps.

  • Open tl-ios-demo.xcworkspace

  • ▶️ Run the project

Tools

SwiftLint

SwiftLint is a tool to enforce Swift style and conventions. SwiftLint enforces the style guide rules that are generally accepted by the Swift community. SwiftLint is installed as part of the development setup. Currently used rules are specified in .swiftlint.yml file. To see lint report while developing you can run or build the project while swift-lint-tl-ios-demo scheme is selected - this will output all of the errors and warnings. We also run SwiftLint & SwiftLint autocorrect (–fix) on git pre-commit (see GitHooks below).

SwiftFormat

In addition to SwiftLint, we use SwiftFormat to further structure the app code. SwiftFormat goes above and beyond what you might expect from a code formatter. In addition to adjusting white space it can insert or remove implicit self, remove redundant parentheses, and correct many other deviations from the standard Swift idioms. SwiftFormat is installed as part of the development setup script. Currently used rules are specified in swiftformatrules file.

Tuist

Tuist is a command line tool that helps you generate, maintain and interact with Xcode projects.

Useful Tuist Commands tuist generate - To generate your project files. This is ran automatically on git checkout hook. Make sure your Xcode is closed when switching the branches as Xcode does not handle branch switching well.

tuist edit - To edit project configuration - Things like adding libraries, packages, setting dependencies etc.
