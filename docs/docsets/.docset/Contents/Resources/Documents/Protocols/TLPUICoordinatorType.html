  • viewStyle

    Controls TrueLayer UI SDK Styling. Please use to change various styling options on the SDK.

    Truelayer UI SDK comes with defaults defined on the init signature. Please see TruelayerUIStyle object for more information.

    Usage 1: You can change individual properies for this to be reflected on the SDK. For example:

       viewStyle.colors.primary = .red

    Usage 2: You can Initialize the whole TruelayerUIStyle object and update it on our SDK . For example:

    let myCustomStyleObject = TruelayerUIStyle()
myCustomStyleObject.colors = TrueLayerUIColors(primaryForeground: .red,
                                     divider: .blue)
viewStyle = myCustomStyleObject

    All of the TruelayerUIStyle colors support dark mode (for iOS 13.0 and newer). TruelayerPaymentsUI SDK exports convenience initializer which allows dual color injection one for light and one for dark apperance mode.

    For example:

       UIColor(lightApperance: UIColor.systemBlue,
           darkApperance: UIColor.black)

    If you specify a singular color then that color will be used by our sdk for all apperance types. Injecting resource colors behaves the same way as programatic colors. If your resource color has Dark Apperance value our SDK will accept that.

  • showProviderSelectionViewController(hostingViewController:modalPresentationStyle:paymentRequest:callback:)

    Use to start Provider Selection flow from TrueLayer Payments UI SDK. Use this when you need your user to select SingleImmediateProvider for their payment.

    Use this when you are operating with the UIViewController stack and want to present on top of the provided hostingViewController. TrueLayerPaymentsUI SDK will present its ViewControllers on top of the supplied UINavigationController.

    Once the Provider Selection operation is finished the TrueLayerPaymentsUI SDK will callback your app with TruelayerProviderSelectionCallback which includes SingleImmediateProvider.

    In addition, the TrueLayerPaymentsUI SDK will dismiss any ViewControllers presented by itself.

    Declaration

    Swift

    func showProviderSelectionViewController(hostingViewController: UIViewController,
                                         modalPresentationStyle: UIModalPresentationStyle,
                                         paymentRequest: TLPPaymentRequest,
                                         callback: TLPProviderSelectionCallback?)

    Parameters

    hostingViewController

    (required) UIViewController on which to present the TrueLayerPaymentsUI SDK ViewControllers.
    modalPresentationStyle

    (required) UIModalPresentationStyle presentation style you would like to enforce e.g. - .automatic, .fullScreen.
    paymentRequest

    (required) TrueLayerPaymentRequest object containing the payment details e.g. - amount and currency.
    callback

    (optional) callback which is called once the provider is selected and confirmed by the user.
  • pushProviderSelectionViewController(hostingNavigationController:paymentRequest:callback:)

    Use to start Provider Selection flow from TrueLayer Payments UI SDK. Use this when you need your user to select SingleImmediateProvider for their payment.

    Use this when you are operating with the UINavigatonController stack. TrueLayerPaymentsUI SDK will push its ViewControllers on top of the supplied UINavigationController.

    Once the Provider Selection operation is finished the TrueLayerPaymentsUI SDK will callback your app with TruelayerProviderSelectionCallback which includes SingleImmediateProvider.

    In addition, the TrueLayerPaymentsUI SDK will pop back to the last visible ViewController from the hostingNavigationController supplied.
